# Tallyo Functional Public-Use Readiness Matrix — 2026-07-18

This is the work-package-1 baseline for `FUNC-READY-001`. It evaluates functional public self-service readiness without changing or overriding the existing legal/public-launch block. Synthetic data is required for every fresh test.

## Interpretation decisions

- The earlier verdict “ready for controlled live use” remains valid evidence, but it is not equivalent to public self-service readiness.
- Direct evidence is reused when the relevant implementation and configuration have not changed. A later, narrower acceptance supersedes an older stale statement.
- The live GBP 1 payment/refund, isolated restore, real-Android MFA recovery and full current-scope regression will not be repeated merely to create a new report.
- Stripe refund receipts were enabled after the completed GBP 1 refund. That test therefore does not prove end-to-end refund-receipt delivery.
- Legal drafting and conclusions remain outside this programme, but the existing legal block and mandatory triggers remain in force.
- Required functional roles are performed sequentially in the current single-agent workflow; this record does not claim concurrent agents are running.
- Production CAPTCHA activation, live money, customer communication, secrets, provider configuration and destructive production actions remain exact action-time Owner gates.
- Performance datasets must be local or isolated synthetic data, not bulk production writes. iPhone Safari is recorded as unavailable if no safe device/service is available rather than simulated falsely.

## Initial functional verdict

**Functionally ready for controlled users.**

Tallyo does not yet meet the supplied definition of **Functionally ready for public self-service use** because public Auth abuse protection is not active, the clean-user journey has not passed as one current acceptance, and dispute, monitoring, communication-status, scale/browser and final synthetic evidence remain incomplete.
